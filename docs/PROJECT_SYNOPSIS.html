2. Introduction

The rapid proliferation of networked devices and the internet has fundamentally transformed how organizations operate, but it has also expanded the attack surface available to malicious actors. According to the IBM X-Force Threat Intelligence Index 2024, the average time from initial network compromise to lateral movement has decreased to under two hours, while the volume of automated attacks continues to grow exponentially. Traditional network defense strategies rely on human-operated Security Operations Centers (SOCs) where analysts manually investigate alerts generated by Intrusion Detection Systems (IDS), assess the severity, and configure firewall rules in response. This manual process typically requires 15 to 30 minutes per incident — a timeframe that is increasingly insufficient against automated attacks that execute in seconds.

Existing IDS platforms such as Snort and Suricata have been the industry standard for signature-based threat detection for over two decades. While these tools excel at identifying known attack patterns through rule matching, they suffer from two critical limitations: they generate alerts without taking any enforcement action, and they cannot detect novel attack patterns that lack known signatures. Research has shown that the average enterprise SOC receives over 10,000 alerts per day, with analysts able to investigate only 4-8% of them, leading to significant threat coverage gaps known as "alert fatigue."

Honeypot systems such as Cowrie (SSH) and T-Pot (multi-service) offer an alternative approach by simulating vulnerable services to capture attacker behavior, including typed commands, attempted credentials, and downloaded malware. However, existing honeypots require static manual deployment before attacks occur and operate independently of the detection system. There is no mechanism for dynamically directing suspicious traffic to honeypot services based on real-time risk assessment.

This project addresses these fundamental gaps by developing NTTH (No Time To Hack), an autonomous network defense system that combines real-time packet capture, hybrid AI-based risk scoring (combining signature-based IDS rules with unsupervised machine learning), autonomous firewall enforcement using Linux kernel-level nftables, and dynamic flow-aware honeypot deployment — all coordinated through an agent-inspired modular pipeline that operates without human intervention. The system additionally integrates wireless monitoring using a commodity Atheros AR9271 USB WiFi adapter in monitor mode for WiFi probe request analysis and deauthentication attack detection, providing convergent wired and wireless threat coverage.

The motivation for this project stems from the observation that while individual components of network defense (detection, ML scoring, firewalling, honeypots) exist in isolation, no existing system integrates all of them into a single closed-loop autonomous pipeline that operates on commodity hardware accessible to small organizations and educational institutions.

3. Problem Statement

3.1 Problem Definition

"Existing Intrusion Detection Systems detect network threats but cannot autonomously respond, leaving a critical 15-30 minute gap between detection and containment during which attackers achieve their objectives."

3.2 Limitations of Existing Solutions

Table 1: Limitations of Existing Approaches
Existing Solution What It Does Critical Limitation
Snort / Suricata Signature-based packet inspection and alerting Alert only — no automated response or enforcement
Cowrie / T-Pot Static honeypot deployment for attacker capture Manually deployed before attacks, not linked to IDS
ML-based IDS Machine learning classification of traffic Produces predictions disconnected from firewalls
AARF (2024) RL-based response strategy optimization Simulation only, no real-world deployment, no latency data
AETHER (2025) AI-generated deception assets Cloud-only, not commodity hardware, no flow-aware redirect

3.3 How NTTH Solves It

NTTH bridges all of these gaps by implementing a complete closed-loop autonomous defense pipeline. The system captures network packets in real-time, scores each packet using a hybrid model combining signature-based IDS rules (60% weight) with unsupervised Isolation Forest machine learning (40% weight), autonomously decides the appropriate response based on risk score thresholds, applies Linux kernel-level nftables firewall rules (block, rate-limit, or redirect), and dynamically deploys honeypot services to capture attacker behavior — all within an average of 127 milliseconds and without any human intervention.

The flow-aware honeypot deployment is a key innovation: instead of statically deploying honeypots and hoping attackers find them, NTTH transparently redirects specific attacker→victim→port flows to appropriate honeypot services (SSH or HTTP) while preserving the attacker's legitimate traffic. This selective deception enables intelligence gathering without disrupting normal network operations.

4. Objectives

  • Objective 1: To design and implement an agent-inspired autonomous pipeline that detects, scores, decides, and enforces network defense actions with sub-second response latency (~127 ms), eliminating the need for manual intervention.
  • Objective 2: To develop a hybrid risk scoring model combining signature-based IDS rules with unsupervised Isolation Forest anomaly detection, validated through systematic ablation study to achieve optimal detection rate (>95%) with minimal false positives (<5%).
  • Objective 3: To implement flow-aware dynamic honeypot deployment that transparently redirects specific attacker-victim-port flows to SSH (Cowrie) and HTTP deception services, capturing attacker commands, credentials, and behavioral patterns.
  • Objective 4: To integrate wireless threat monitoring using a commodity Atheros AR9271 USB WiFi adapter in monitor mode for WiFi probe request analysis, deauthentication attack detection, and device presence tracking.
  • Objective 5: To develop a real-time cross-platform dashboard (Flutter Web + Android) that visualizes threats, firewall rules, honeypot sessions, and network topology via WebSocket live updates.

5. Scope of the Project

5.1 What the System Will Do

  • Capture and analyze all IP packets on a local area network in real-time
  • Detect port scans, SYN floods, brute force attacks, and HTTP probing
  • Score threats using hybrid rule-based + ML anomaly detection
  • Autonomously apply firewall rules (block, rate-limit, redirect) via nftables
  • Dynamically redirect attackers to SSH and HTTP honeypots
  • Capture attacker commands, credentials, and session duration
  • Monitor WiFi probe requests and deauthentication attacks
  • Display everything on a real-time Flutter dashboard

5.2 What the System Will Not Do

  • Deep packet inspection of encrypted (TLS/SSL) payload content
  • Enterprise-scale multi-site distributed deployment (single-host scope)
  • 5 GHz / 6 GHz WiFi monitoring (AR9271 is 2.4 GHz only)
  • Attack attribution beyond GeoIP location data
