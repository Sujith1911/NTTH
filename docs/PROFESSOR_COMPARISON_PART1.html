Section 1: Initial Idea vs. Where We Are Now

1.1 The Initial Proposal (What We Pitched)

When we first proposed this project, our idea was straightforward on paper — build a smart gateway that can watch over a home network and automatically protect IoT devices from attacks. We envisioned a system that could:

  • Sit as an Access Point so all device traffic flows through us
  • Detect common attacks like port scans and brute-force login attempts
  • Use some form of machine learning to catch unusual behavior
  • Redirect attackers to fake servers (honeypots) instead of real devices
  • Block or isolate dangerous devices automatically
  • Show everything on a dashboard in real time

Honestly, at the proposal stage, we had a conceptual understanding of each piece but hadn't figured out how they would all connect. The proposal described an "Observe–Analyze–Decide–Act" pipeline, but the actual architecture — the event bus, the agent stages, the NAT redirection logic — none of that existed yet. It was an idea, not a design.
